데이터 저장 및 삭제

어디에 무엇이 저장되고, 백업과 초기화가 어떤 범위에 적용되는지 설명합니다.

최종 업데이트: 2026년 7월 15일

요약: 일정 데이터는 현재 브라우저에만 저장되며 다른 브라우저나 기기로 자동 동기화되지 않습니다.

백업 파일에 포함되는 내용

백업 다운로드를 누르면 일정, 빠른 타이머, 테마, 공휴일 선택 상태와 백업 생성 시각을 JSON 파일로 만듭니다. 파일은 사용자의 다운로드 폴더 등 브라우저가 지정한 위치에 저장됩니다.

  • 백업 파일은 운영자 서버에 저장되지 않습니다.
  • 메모와 일정명이 포함되므로 다른 사람에게 전달하기 전에 내용을 확인해야 합니다.
  • 백업 불러오기를 완료하면 현재 일정과 설정이 백업 내용으로 교체됩니다.
  • 파일을 불러오는 것만으로는 적용되지 않으며 확인 화면에서 복원을 완료해야 합니다.

서비스 안에서 초기화하기

  1. 1

    필요하면 먼저 백업합니다

    알람 목록의 백업 다운로드를 눌러 현재 데이터를 파일로 보관합니다.

  2. 2

    전체 초기화를 누릅니다

    알람 목록 오른쪽의 전체 초기화를 누르면 확인 창이 열립니다.

  3. 3

    삭제 범위를 확인합니다

    저장된 일정과 실행 대기 타이머, 공휴일 선택이 삭제되고 빠른 타이머는 기본값으로 돌아갑니다.

전체 초기화 뒤에도 남는 항목: 선택한 라이트·다크 테마와 브라우저 알림 권한은 유지됩니다.

브라우저에서 완전히 지우기

테마를 포함한 사이트 저장 정보까지 지우려면 브라우저 설정에서 alarm-schedule.pages.dev의 사이트 데이터 또는 저장 공간을 삭제합니다. 메뉴 이름과 위치는 브라우저에 따라 다릅니다.

알림 권한 삭제

주소창의 사이트 설정 또는 브라우저 개인정보 설정에서 알림 권한을 초기화합니다.

백업 파일 삭제

다운로드한 JSON 파일은 사이트 초기화와 무관하므로 기기의 파일 관리 기능으로 직접 삭제합니다.

다른 브라우저 데이터

Chrome에서 삭제해도 Edge나 Firefox에 저장된 데이터는 함께 삭제되지 않습니다.

문의 폼 데이터

Google 문의 폼으로 제출한 내용은 브라우저 초기화 대상이 아니며 별도 삭제 요청이 필요합니다.

복구할 수 없는 경우

브라우저 사이트 데이터를 삭제했거나 전체 초기화를 완료한 뒤 백업 파일도 없다면 서비스에서 이전 일정을 복구할 수 없습니다. 기기를 변경하거나 브라우저를 초기화하기 전에는 백업 파일을 내려받아 별도로 보관해야 합니다.
